## What is a monorepo

A _monorepo_ is a software development strategy where code for many projects are stored in the same repository

- Libraries / packages
- Multiple artifacts
  - Business Logic
  - React components
  - Web Service

---

## Monorepo &ne; Monolith

- A monorepo is about scope, not deployment
- A monorepo can still have it's multiple components and _deliverables_

---

## Advantages

- Better defined boundaries
- Simplified _onboarding_
- Simpler dependency management
- Version management

---

## Who uses it

- jest

## Drawbacks

- Larger repository ..
- More branches/tags
- CI/CD can be more complex
